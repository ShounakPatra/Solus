# Security Policy

## Supported Versions

Security fixes are provided for the latest published Solus release.

| Version | Supported |
|---|:---:|
| Latest release | Yes |
| Older releases | No |

## Reporting a Vulnerability

Please do not disclose security vulnerabilities in a public issue, discussion, or pull request.

Use [GitHub private vulnerability reporting](https://github.com/ShounakPatra/Solus/security/advisories/new) to send the report directly to the maintainer. Include:

- A clear description of the vulnerability and its impact
- The affected Solus version, Android version, device, model, and runtime
- Reproduction steps or a minimal proof of concept
- Relevant logs or screenshots with tokens and personal data removed
- Any suggested mitigation, if known

You should receive an acknowledgement within seven days and a status update within fourteen days. Please allow time for a fix and coordinated release before public disclosure.

## Security Scope

Reports about Solus source code, local data handling, model downloads, file parsing, and on-device runtime integration are in scope. Vulnerabilities in third-party model files or hosting services should also be reported to their respective publishers.
